## 3. 接口测试 (API Testing)

主要针对 Backend Controller 层提供的 RESTful 接口进行验证。

### 3.1 核心接口清单

- **GET /api/textbooks**: 获取教材列表，支持分页、搜索。
- **POST /api/textbooks**: 创建新教材（需鉴权）。
- **POST /api/orders**: 创建订单（事务一致性测试重点）。
- **POST /api/users/[id]/balance**: 余额变更接口（需严格校验金额）。
- **GET /api/users/[id]**: 获取用户详情及信用分。

### 3.2 重点测试场景 (JSON Payload)

**场景：余额充值**
```json
POST /api/users/{id}/balance
Content-Type: application/json
{
  "amount": 50.00
}
```
*预期*: 200 OK, 返回更新后的 User 对象，balance 字段增加 50。

**场景：订单创建**
```json
POST /api/orders
Content-Type: application/json
{
  "textbookId": "tb_123",
  "buyerId": "user_456",
  "addressId": "addr_789"
}
```
*预期*: 200 OK, 订单创建成功，Textbook 状态更新，Buyer 余额扣除，Seller 余额增加（或冻结）。

## 4. 性能与安全测试 (Performance & Security)

### 4.1 性能关注点
- **加载速度**: 首页及教材列表页在 1000+ 数据量下的加载时间（目标 < 1.5s）。
- **图片加载**: 检查图片是否启用懒加载及尺寸优化。
- **并发处理**: 模拟多用户同时购买同一本教材，验证库存锁/事务一致性（防止超卖）。

### 4.2 安全关注点
- **越权访问**: 验证非管理员是否能访问 `/admin` 页面或接口。
- **数据篡改**: 验证修改教材/订单接口是否校验了 `userId`（只能修改自己的数据）。
- **输入验证**: 验证金额输入框是否拦截负数、非数字字符及超长小数（如 `0.230000004` 问题）。

## 5. 测试总结与建议

### 5.1 当前版本已知问题
1. **数值精度显示**: 曾出现浮点数显示过长问题（如 0.230000...4），已在 v0.1.0 修复，需回归测试所有金额输入框。
2. **支付流程**: 当前为模拟余额支付，无真实网关对接，测试时需注意区分环境。

### 5.2 最近修复 (v0.1.2)
1. **编辑功能上线**: 支持对已发布教材的二次编辑，解决了信息填错需重新发布的痛点。
2. **上下架管理**: 新增卖家自主下架功能，优化了商品生命周期管理。
3. **租赁模式优化**: 租赁模式下“出售价格”改为选填，并移除了必填星号标记，减少用户困惑。
4. **UI 体验升级**: “我的发布”列表页重构为左图右文布局，提升信息阅读效率；商品详情页状态文案修正为“商品已下架”。

### 5.3 改进建议
- 建议引入单元测试框架 (Jest/Vitest) 对核心计算逻辑（如订单金额、积分换算）进行覆盖。
- 建议增加 E2E 测试 (Playwright/Cypress) 自动化覆盖核心交易链路。
